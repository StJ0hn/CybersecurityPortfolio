# Security Audit

## Description

Security audit activity from the **Google Cybersecurity Certificate**, covering scope definition, risk assessment, security controls assessment, and a compliance checklist for a fictional organizational scenario.

## Overview

This activity simulates a security audit for a fictional organization, applying the core audit workflow: comparing what security practices *should* be in place (based on established frameworks and regulations) against what is *actually* implemented, in order to identify gaps and prioritize remediation.

The deliverable follows four sections: audit scope and goals, a risk assessment of current assets, a controls assessment categorized by type (administrative, technical, physical), and a compliance checklist against applicable regulations.

## Topics

- Security audit scope and goal definition
- Risk assessment and risk scoring
- Security controls: administrative, technical, physical
- Regulatory compliance: GDPR, PCI DSS

## Learning Objectives

- Define the scope and goals of a security audit aligned with an established framework (NIST CSF)
- Assess organizational assets and score associated risks based on potential impact
- Categorize and prioritize security controls by type and implementation status
- Evaluate compliance requirements against organizational data handling practices
- Translate a course activity into a documented, portfolio-ready artifact
